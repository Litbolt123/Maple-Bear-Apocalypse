## Summary

Performance pass for **day 0–1** and **village / new-chunk** load, plus **buff bear** conversion and death fixes. Tested in-world on this build.

---

## Performance

- **Day 0–1 work spread** (`mb_workSpread.js`): expensive loops run on longer intervals and stagger across players, dimensions, and 32-block grid cells instead of all at once.
- **Entity queries:** mob cache, bear snapshots, spawn metrics, and related scans use **player-anchored sections** (3×3 cells) instead of huge radius or full-dimension sweeps where possible.
- **Spawn / village path:** spawn controller bear counts use snapshot cache; item metrics and biome fallback throttled; dimension adaptation no longer scans every entity in a dimension each tick.

If a world still feels heavy after day 2, use **Journal → Performance** (spawn load, adaptive perf, LAGGY tier).

---

## Buff bears & conversions

- **Death explosion:** buff bears again leave a **powder/block burst** on death (same family as stuck fuse). Fresh mob-conversion spawns are excluded so they do not instantly pop.
- **Mob kills by buff bears:** conversion respects **victim size** — tiny → tiny Maple Bear, normal → infected for the current day, large → buff — not always `mb:mb_day00`.
- **Universal buff cap:** all conversion and spawn paths share the same limit (**1 / 2 / 3** buff bears by player count in the dimension). Over cap → **infected** tier instead of more buffs.
- **Storm fixes:** no **double conversion** when a Maple Bear killed the mob; storm mass-convert respects the buff cap **dimension-wide** (solo worlds no longer get a full wave of buffs).

## Install / upgrade

1. Remove or disable the previous beta pack from the world.
2. Unzip **`BP`** and **`RP`** from the release zips (or export `.mcpack` from Bridge using **release** config).
3. Enable both packs on the world; restart if the client cached old scripts.
